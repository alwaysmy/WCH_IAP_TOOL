## 3. 新增功能设计

### 3.1 文件路径历史记录功能

#### 3.1.1 需求描述

改进文件路径保存功能，支持历史路径记录和下拉选择。

#### 3.1.2 功能特性

| 特性 | 说明 |
|------|------|
| 保存多个历史路径 | 每次使用新路径换行保存到txt文件 |
| 下拉列表选择 | 文件路径框改为ComboBox（允许手动编辑） |
| 自动加载最后路径 | 打开程序时自动加载最后使用的路径 |
| 历史路径选择 | 通过下拉列表选择之前使用过的路径 |

#### 3.1.3 文件格式

**文件名：** `filehistory.txt`（与exe同目录）

**内容格式：**
```
D:\Projects\firmware1.hex
D:\Projects\firmware2.hex
D:\Projects\firmware3.hex
```

- 每行一个路径
- 最新的路径在最后一行
- 最多保存20条，超出时删除最早的

#### 3.1.4 数据结构

```csharp
private const string HistoryFileName = "filehistory.txt";
private const int MaxHistoryCount = 20;
private List<string> fileHistory = new List<string>();
```

#### 3.1.5 功能模块

**1. 界面修改**
- 将 `TextBox filePathTextBox` 改为 `ComboBox filePathComboBox`
- 设置 `DropDownStyle = ComboBoxStyle.DropDown`（允许编辑和下拉）

**2. 加载历史记录 `LoadFileHistory()`**
- 读取文件所有行
- 验证路径有效性（文件是否存在）
- 加载到 fileHistory 列表
- 填充到 ComboBox
- 自动选择最后一项

**3. 保存历史记录 `SaveFileHistory(string newPath)`**
- 如果路径已存在，先移除（避免重复）
- 添加到列表末尾
- 如果超出最大数量，移除最早的
- 写入文件（每行一个路径）

**4. 事件处理**
- `SelectedIndexChanged`：用户从下拉列表选择
- `Leave`：用户手动输入完成，验证路径

#### 3.1.6 修改点汇总

| 文件 | 位置 | 修改内容 |
|------|------|----------|
| Form1.cs | CreateDownloadSection() | TextBox → ComboBox |
| Form1.cs | InitializeUI()末尾 | 调用 LoadFileHistory() |
| Form1.cs | FormClosing事件 | 改为保存历史而非单一路径 |
| Form1.cs | FileSelectButton_Click() | 选择后添加到历史 |
| Form1.cs | DownloadProgram() | 下载成功后保存历史 |
| Form1.cs | 新增方法 | LoadFileHistory(), SaveFileHistory() |

#### 3.1.7 异常处理

- 文件读取失败：忽略，使用空列表
- 路径无效：不添加到历史
- 文件写入失败：静默处理，不影响程序运行

#### 3.1.8 兼容性

- 保留文件拖放功能
- 保留 `lastfile.txt` 或迁移到新文件（建议统一使用 `filehistory.txt`）

## 3. 代码改进建议

### 3.1 代码重构

**问题：** CLI和GUI存在重复代码

**建议：** 创建共享库
```
WCHIAPTool/
├── WchHexToBinConverter.cs    # 已实现
├── WchIapProtocol.cs          # 新增：IAP协议封装
├── WchDeviceManager.cs        # 新增：设备管理
├── WCHIAPToolCLI/
└── WCHIAPToolNew/
```

### 3.2 错误处理改进

**当前：** 简单的try-catch

**建议：** 自定义异常类
```csharp
public class WchIapException : Exception
{
    public byte ErrorCode { get; set; }
    public string ErrorDescription { get; set; }
}
```

### 3.3 异步处理

**问题：** 下载时界面卡顿

**建议：** 使用async/await
```csharp
private async Task DownloadProgramAsync()
{
    await Task.Run(() => DownloadProgram());
}
```
