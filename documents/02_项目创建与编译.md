# 项目创建与编译

## 1. 环境准备

### 1.1 安装.NET SDK

本项目使用.NET 9.0，需要先安装.NET SDK。

**检查是否已安装：**
```powershell
dotnet --version
```

**下载地址：** https://dotnet.microsoft.com/download

### 1.2 项目依赖

- **CH375DLL64.dll**：USB驱动库
  - 需要复制到项目输出目录
  - 位置：`CH375_CH372_lib/CH375DLL64.dll`

## 2. 创建项目

### 2.1 CLI项目创建

```powershell
# 创建控制台项目
dotnet new console -n WCHIAPToolCLI

# 进入项目目录
cd WCHIAPToolCLI

# 设置平台目标为x64
# 编辑.csproj文件，添加：<PlatformTarget>x64</PlatformTarget>
```

### 2.2 GUI项目创建

```powershell
# 创建Windows Forms项目
dotnet new winforms -n WCHIAPToolNew

# 进入项目目录
cd WCHIAPToolNew

# 设置平台目标为x64
# 编辑.csproj文件，添加：<PlatformTarget>x64</PlatformTarget>
```

### 2.3 项目配置文件

**CLI项目 (WCHIAPToolCLI.csproj)：**
```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net9.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>
    <Nullable>enable</Nullable>
    <PlatformTarget>x64</PlatformTarget>
  </PropertyGroup>

  <ItemGroup>
    <Compile Include="..\WchHexToBinConverter.cs" Link="WchHexToBinConverter.cs" />
  </ItemGroup>
</Project>
```

**GUI项目 (WCHIAPToolNew.csproj)：**
```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>WinExe</OutputType>
    <TargetFramework>net9.0-windows</TargetFramework>
    <Nullable>enable</Nullable>
    <UseWindowsForms>true</UseWindowsForms>
    <ImplicitUsings>enable</ImplicitUsings>
    <PlatformTarget>x64</PlatformTarget>
  </PropertyGroup>

  <ItemGroup>
    <Compile Include="..\WchHexToBinConverter.cs" Link="WchHexToBinConverter.cs" />
  </ItemGroup>
</Project>
```

### 2.4 添加共用文件

两个项目都链接了 `WchHexToBinConverter.cs` 文件，实现代码复用。

## 3. 编译项目

### 3.1 编译CLI版本

```powershell
dotnet build .\WCHIAPToolCLI\WCHIAPToolCLI.csproj
```

**输出位置：** `WCHIAPToolCLI\bin\Debug\net9.0\WCHIAPToolCLI.exe`

### 3.2 编译GUI版本

```powershell
dotnet build .\WCHIAPToolNew\WCHIAPToolNew.csproj
```

**输出位置：** `WCHIAPToolNew\bin\Debug\net9.0-windows\WCHIAPToolNew.exe`

### 3.3 编译所有项目

```powershell
dotnet build
```

## 4. 运行项目

### 4.1 CLI版本

```powershell
# 直接运行
dotnet run --project .\WCHIAPToolCLI\WCHIAPToolCLI.csproj

# 或运行编译后的exe
.\WCHIAPToolCLI\bin\Debug\net9.0\WCHIAPToolCLI.exe
```

### 4.2 GUI版本

```powershell
# 直接运行
dotnet run --project .\WCHIAPToolNew\WCHIAPToolNew.csproj

# 或运行编译后的exe
.\WCHIAPToolNew\bin\Debug\net9.0-windows\WCHIAPToolNew.exe
```

## 5. 复制依赖文件

编译后需要确保 `CH375DLL64.dll` 在输出目录中：

```powershell
# 复制到CLI输出目录
copy CH375_CH372_lib\CH375DLL64.dll WCHIAPToolCLI\bin\Debug\net9.0\

# 复制到GUI输出目录
copy CH375_CH372_lib\CH375DLL64.dll WCHIAPToolNew\bin\Debug\net9.0-windows\
```

**注意：** 项目中已包含该DLL文件，会自动复制到输出目录。

## 6. 发布项目

### 6.1 发布为单文件

```powershell
# CLI版本
dotnet publish .\WCHIAPToolCLI\WCHIAPToolCLI.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true

# GUI版本
dotnet publish .\WCHIAPToolNew\WCHIAPToolNew.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true
```

**输出位置：**
- CLI: `WCHIAPToolCLI\bin\Release\net9.0\win-x64\publish\`
- GUI: `WCHIAPToolNew\bin\Release\net9.0-windows\win-x64\publish\`

### 6.2 发布为依赖框架版本

```powershell
dotnet publish .\WCHIAPToolNew\WCHIAPToolNew.csproj -c Release
```

### 6.3 关于 Native AOT 发布

**重要提示：Windows Forms 不支持 Native AOT**

尝试使用 Native AOT 发布时会遇到以下错误：
```
error NETSDK1175: 启用剪裁时，不支持或不推荐使用 Windows 窗体。
error PublishTrimmed is implied by native compilation and cannot be disabled.
```

**原因：**
- Native AOT 强制启用裁剪（Trimming）
- Windows Forms 不支持裁剪（.NET 10 SDK 明确限制）
- 两者冲突，无法同时满足

**替代方案：**

| 方案 | 命令 | 文件大小 | 运行时要求 |
|------|------|----------|------------|
| **自包含单文件（推荐）** | `dotnet publish -r win-x64 --self-contained true -p:PublishSingleFile=true` | 约 50MB | 无需安装运行时 |
| **依赖框架** | `dotnet publish` | 约 100KB | 需要安装 .NET 9 运行时 |

**结论：** 对于 Windows Forms 应用程序，建议使用**自包含单文件发布**而非 Native AOT。

## 7. 常见问题

### 7.1 编译错误：找不到CH375DLL64.dll

**解决：** 确保DLL文件已复制到项目目录，并在.csproj中配置复制：

```xml
<ItemGroup>
  <None Update="CH375DLL64.dll">
    <CopyToOutputDirectory>PreserveNewest</CopyToOutputDirectory>
  </None>
</ItemGroup>
```

### 7.2 运行错误：BadImageFormatException

**原因：** 32位/64位不匹配

**解决：** 确保项目设置为x64平台：
```xml
<PlatformTarget>x64</PlatformTarget>
```

### 7.3 运行错误：找不到入口点

**原因：** DLL函数名不匹配

**解决：** 检查DllImport的EntryPoint是否正确：
```csharp
[DllImport("CH375DLL64.dll", EntryPoint = "CH375OpenDevice")]
```
